Microsoft Entra ID

Microsoft Entra ID is Microsoft’s cloud identity platform. It answers who you are and what you can access - often in a few hundred milliseconds.

Three years ago, a colleague called me in a mild panic… She’d read docs and watched videos and felt more confused. “What does it actually do… like, in real life?” That’s where we’ll start.

📌 So what is it, actually?

Microsoft Entra ID is a cloud-based identity and access management service. Break that sentence down and you get everything you need:

Identity management means Entra ID keeps track of who exists in your organization - employees, contractors, guests, apps, and devices. Each one gets a digital record.

Access management means Entra ID decides what each identity can do. Can this user open SharePoint? Can that app read emails? Can this laptop access VPN? Entra ID is the system making those calls.

The two questions it answers every time something tries to connect: Who are you? and What are you allowed to do?

💡 Why this matters to you as a beginner

A common beginner mistake: thinking Entra ID is “just a user list.” It isn’t. It’s a ⚙️decision engine. Every sign-in checks credentials, evaluates risk signals, applies security policies, and issues a token - fast.

If a user can’t access an app, the fix isn’t always “reset password.” It might be 🧩Conditional Access, missing group membership, a license not assigned, or a non‑compliant device.

📅 What it handles day‑to‑day

🪪Authentication is verifying identity: password, phone prompt, fingerprint - Entra says yes/no.

🚪Authorization happens right after: once Entra knows who you are, it determines what you can access. It issues a 🎫token - a short‑lived digital pass.

Device integration means Entra can consider whether a device is compliant. A laptop without disk encryption might be blocked from sensitive apps even if credentials are fine.

🧩Conditional Access is the security logic layer: evaluates context (where, device, risk) and decides allow, block, or require MFA.

💭 The name change confusion (let’s clear this up)

Entra ID and Azure Active Directory are the same thing. Microsoft rebranded Azure AD to Microsoft Entra ID in 2023. Older docs and tutorials still say “Azure AD.” Same underlying tenant and technology - new name.

You’ll still hear experienced admins say “Azure AD” out of habit. Totally normal.

📖 A real implementation story

Early in my career, I worked with a 400‑person company moving from a legacy VPN setup to Entra ID‑based access. Users had multiple passwords across systems, and password resets were a constant drain.

After setting up Entra ID with Single Sign‑On, users had one identity. One sign‑in got them into Teams, SharePoint, Salesforce, and internal apps. Password reset tickets dropped by ~60% in the first month.

Not because passwords got “easier,” but because there was only one to manage - and policies applied consistently.

💡 Why this is your starting point

Every other identity concept - Conditional Access, device compliance, roles, MFA, hybrid identity - connects back to Entra ID as the foundation.

Start here. Get this solid. Everything else builds on it. 🔑
